# Slide 13 — How LLMs "Listen"
- **Archetype**: chat-log
- **Grid & layout**: Two vertically-stacked, nested dashed-border frames on a full-bleed dark canvas (slide 13), NOT side-by-side columns. Upper frame is large (spans most of the width, top ~60%); lower frame is smaller and sits below/inside the lower portion, horizontally centered and narrower. Each frame carries a small centered tab-label straddling its top edge. Right-side gutter of the upper frame holds a vertical column of 4 emoji+label side-annotations aligned to rows of the prompt text. Footer at bottom.
- **Type roles**: head — frame tab-labels (小, one CJK phrase each). Inside upper frame: a mono/monospace English block (System Prompt lines, code-like, evenly tracked) — this is the only MONO role in the range so far. zh+en heading rows (系統指令 System Prompt：, 使用者指令 User Prompt：) mix bold CJK + Latin. Side annotations are zh (角色定位 / 安全紅線 / 法律與版權 / 記憶與隱私) with emoji. Lower frame contains a chat-input widget with small UI labels.
- **Color / tokens**: Near-black background with a faint greenish grid/glow in the upper area. Two accent border tokens: the OUTER upper frame border is LIME dashed, the INNER lower frame border is PURPLE/violet dashed — lime used here as a BORDER (contrast with slide 10's lime-as-text). Tab labels colored to match their frame accent (lime label on the lime frame, purple label on the purple frame). Mono code block in light grey on a dark inset panel; chat input on a dark rounded surface; a small ORANGE send button (arrow) bottom-right of the input; a grey "Sonnet 4.6" model-selector chip. No viridis.
- **Components present**:
  - Outer lime dashed frame, tab-label 模型看到的 ("what the model sees").
  - Inner purple dashed frame, tab-label 用戶看到的 ("what the user sees").
  - System Prompt panel: 💥-like burst emoji + heading 系統指令 System Prompt：, then a mono multi-line English block with vertical ⋮ ellipses between excerpted lines (The assistant is Claude…; …not write malicious code…; Paraphrasing…IP.; …selective applies memories…).
  - Right-gutter side-annotations (4, vertical): emoji + zh label — 🧑‍🎨-like 角色定位, 🛡️-like 安全紅線, ⚖️ 法律與版權, 🧠 記憶與隱私 (some emoji appearance unsure).
  - User Prompt block: 👤-like emoji + heading 使用者指令 User Prompt：, then a chat-input widget containing the line 幫我統整我附上的這篇文章, a "+" attach affordance, a "Sonnet 4.6" model selector with dropdown caret, and an orange circular send/arrow button.
- **Motion hints**: The nested frames (model-view wrapping, user-view inset) suggest a reveal that peels from user-facing to model-facing context; not confirmable.
- **Reusable pattern**: This slide is an instance of a nested annotated-frame explainer contrasting "what the model sees" (mono system prompt) vs "what the user sees" (chat input), with emoji side-labels.
- **One-offs / inconsistencies**: Bespoke composite — the only mono/code block and the only realistic chat-UI widget (Sonnet 4.6 chip, orange send button) in the range; likely not a repeatable component. Two distinct dashed-border accent colors (lime vs purple) is unique here. Not comparison-columns despite the two-view contrast because the frames are stacked/nested, not columnar. Several emoji glyphs are ambiguous (marked unsure).

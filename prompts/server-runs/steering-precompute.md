# steering — compute the concept directions + bake the real presets

Replaces the hand-authored sample in
`apps/course2/public/data/course2/steering/{presets,features}.json` with
RECORDED real Qwen3-0.6B outputs, installs the direction vectors for the live
server, and turns on `/steering/generate`.

## 1. What it produces

| Artifact | Where | Committed? |
| -------- | ----- | ---------- |
| `steering/directions.npz` — 4 unit directions + layer + absolute scales | `precompute/artifacts/steering/` | NO (gitignored server state) |
| `steering/presets.json` — base + feature×strength replies for the 3 preset prompts | `apps/course2/public/data/course2/steering/` | yes (small JSON) |
| `steering/features.json` — knob catalog (中文 label, gloss, pole labels, layer, scale) | `apps/course2/public/data/course2/steering/` | yes (small JSON) |
| manifest updates (`steering-presets`, `steering-features` ids, real bytes) | `apps/course2/public/data/course2/manifest.json` | yes |

Checkpoint / feature source (the resolved sourcing decision): the base is the
already-served **Qwen/Qwen3-0.6B** (same instance as next-token / transformer /
lora — no second base model). Features are **contrastive activation-addition
steering vectors** (mean residual-stream difference at layer 14 over
hand-written 中文 contrast sentence sets in
`precompute/src/camp_precompute/steering.py`) — NOT a public SAE. Chosen
because 中文-labeled hand-picked directions beat SAE releases (English
auto-labels; GPT-2/Gemma bases answer poorly in 正體中文) on legibility for
high-schoolers, and because it adds zero VRAM and zero training. Computing the
4 directions is ~80 short forward passes — under a minute on a V100.

## 2. Prereqs

- The base deploy from `server/README.md` is in place (repo cloned,
  `server/.venv` synced, systemd replicas running).
- `git pull` so the tree has the `steering-vectors` / `steering` cli
  subcommands + router. No new Python dependencies — no `uv sync` needed; if
  you refresh `server/.venv` anyway, it must be `uv sync --extra gpu` (plain
  `uv sync` drops the diffusion deps; see the server-runs README).

## 3. The commands

Run precompute through the SERVER venv so the recorded presets use the exact
torch/transformers the live server answers with:

```bash
cd ~/sitcon-camp-2026-ml/precompute

# 3a. Compute the 4 concept directions (forward passes only, no training;
#     under a minute). Writes precompute/artifacts/steering/directions.npz.
uv run --project ../server camp-precompute steering-vectors

# 3b. Bake the presets: real base + each feature over the 3 preset prompts at
#     strengths {-2, -1, +1, +2} (greedy). Overwrites the hand-authored sample
#     and updates manifest.json.
uv run --project ../server camp-precompute steering
```

## 4. Verify

```bash
# generator must now be "camp-precompute steering", not "…steering-sample":
head -3 ../apps/course2/public/data/course2/steering/presets.json

# spot-check the reveal: boba at +2 for 描述你最喜歡的地方 should obsess
# about 珍珠奶茶; the same prompt's base should not mention it
python3 -c "import json;d=json.load(open('../apps/course2/public/data/course2/steering/presets.json'));print(d['base']['描述你最喜歡的地方']);print('---');print(d['outputs']['boba']['描述你最喜歡的地方'][3])"

# features.json scale must be a real number now (null in the dev sample)
python3 -c "import json;d=json.load(open('../apps/course2/public/data/course2/steering/features.json'));print([(f['id'],f['scale']) for f in d['features']])"

git diff --stat ../apps/course2/public/data/course2/
```

**Tuning.** If a feature reads flat at ±2, or degenerates into repetition /
non-language, adjust in `precompute/src/camp_precompute/steering.py` and re-run
3a + 3b. Lessons already baked in from the 2026-07-09 tuning session (see the
module docstring for detail):

- Per-feature `boost` on the FeatureSpec is the strength dial (COEFF stays
  0.08 globally). Flat → raise the feature's boost; repetition/degeneration →
  lower it. Legible effects start around boost 2 (≈16-20% of the residual
  norm at +2) and text breaks past roughly boost 4.5.
- A concept knob that surfaces the wrong entity usually means the MODEL
  doesn't know the concept — knowledge-check with 「介紹一下X」 before blaming
  the direction (Qwen3-0.6B doesn't know 金門大橋-as-Golden-Gate or 熊貓;
  that's why the knobs are 珍珠奶茶/貓/英文/心情).
- `STEER_LAYER` (default 14 of 28) — earlier reads subtler, later gets
  assertive but brittle. Sweeping 8–20 didn't beat 14 for these features.
- A single weak feature usually means its contrast sets are too close — sharpen
  the sentences (more of the concept in `pos`, none of it in `neg`). Concept
  knobs set `span` so only the concept word's tokens enter the mean.

The 白話文 knob labels live in the same file; the shipped features.json is
regenerated by 3b, so label edits need no frontend change.

## 5. Deploy

```bash
# commit ONLY the small JSON (directions.npz is gitignored)
cd ~/sitcon-camp-2026-ml
git add apps/course2/public/data/course2/steering apps/course2/public/data/course2/manifest.json
git commit -m "data: real steering directions + presets from the V100 box"
git push

# restart every replica so loader.py picks up directions.npz
sudo systemctl restart camp-server@0 camp-server@1 camp-server@2 camp-server@3

# health: startup log must show "steering: loaded directions (layer 14): boba, cat, english, mood"
journalctl -u camp-server@0 -n 20 | grep steering

# smoke test (after /auth per server/README.md §5; expect the same text as
# presets.json's outputs.boba['介紹一下你自己'][3]):
curl -sb "$JAR" -H 'Content-Type: application/json' \
  -d '{"prompt":"介紹一下你自己","features":[{"id":"boba","strength":2}]}' \
  http://127.0.0.1:8300/steering/generate
```

Frontend: redeploy (Vercel picks up the committed JSON). The station then
serves presets offline (every single-knob slider stop), and live generation for
typed prompts and multi-knob combinations.

## Notes / knobs

- `/steering/generate` holds the per-process `lm_lock` for the whole greedy
  generation (≤64 new tokens, a few seconds on V100 fp32) — same cost profile
  as `/lora/generate`; the existing concurrency slot + queue keep a classroom
  burst orderly.
- The hook is added and removed per request inside the lock
  (`camp_precompute.steering.steered`), so no other Qwen route can ever see a
  steered residual stream.
- Multi-knob requests sum the deltas. The schema caps each strength at ±2 and
  the list at 8 entries; `steered.__init__` additionally clips the TOTAL delta
  norm to 1.2 × the strongest single knob at full strength, so combos degrade
  gracefully and single-knob presets are never clipped (live == baked).
